# Glossary

This glossary explains the main terms you will see in Hooklusion's user docs.

## Animation

A named playback unit made from one or more image frames plus timing information.

## Animation Studio

The built-in editor where you preview, import, edit, save, and apply character profiles.

## Apply

Tell Hooklusion to start using a profile in actual app behavior, not just inside preview.

## Bundled Profile

A profile that ships with the app by default.

Examples include `gpchan-default` and `office-assistant-default`.

## Default Profile

The profile Hooklusion uses by default unless another choice overrides it.

## Frame

A single image inside an animation sequence.

## Frame Weight

A relative timing value that affects how long one frame is shown compared with the others in the same animation.

## Interaction State

A state caused by direct interaction with the character window, such as click or drag.

## Profile

A character definition that includes its animations, state mappings, and presentation behavior.

## Prompt Received

The state Hooklusion uses when your LLM input has been received.

## Save

Write your current profile edits to disk so they are not lost.

## Sprite Set

A directory of PNG frames organized into Hooklusion's expected folder structure so it can be imported into a profile.

## State

A named situation Hooklusion can represent, such as `thinking`, `tool_active`, or `done`.

## Thinking

The state used between hook events when the app interprets the agent as waiting or reasoning.

## Tool Active

The generic work-in-progress state used when a more specific tool state is not available.

## Transition In / Transition Out

Optional animations used when changing profiles so the switch feels smoother.
